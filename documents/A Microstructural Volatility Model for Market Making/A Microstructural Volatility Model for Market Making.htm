Measures of Microstructural Volatility

We went over this briefly in the previous article.
We mentioned the following metrics:

  • std of executed trades prices

  • Volume

  • std of price impact

  • biggest absolute price impact

  • std of midprice change

  • std of spread (and its varients) change

  • Price Range: Biggest midprice - Smallest midprice (or other price measure)

We are gonna be using the std of midprice change in this article.
Whenever midprice changes a lot we expect “fair price” (whatever that actually is) to be changing around a lot as well.
So we are using midprice as a proxy for fair price here.

If you already have a model for fair price that you quote around then you can (and should) instead use the volatility of your own fair price.

Let’s, as usual, load in our data first:

data = pd.read_csv("binance-futures_quotes_2024-08-01_BTCUSDT.csv").set_index("timestamp")
data.index = pd.to_datetime(data.index*1000)

And now we calculate midprice:

data['mid'] = (data['ask_price'] + data['bid_price']) / 2

And finally the volatility using a rolling 1 minute window:

data['vol'] = data['mid'].rolling("1Min").std()

Baseline Market Making Model

Before we continue with the volatility work we are just gonna create a very, very simple MM model: Fixed distance quoting around midprice, updated once a second.

spread = 0.0004 # 4bps

data['MM_bid'] = data['mid'] * (1 - spread/2)
data['MM_ask'] = data['mid'] * (1 + spread/2)

data['MM_bid'] = data.groupby([data.index.date, data.index.hour, data.index.minute, data.index.second])['MM_bid'].transform('first')
data['MM_ask'] = data.groupby([data.index.date, data.index.hour, data.index.minute, data.index.second])['MM_ask'].transform('first')

Now it’s time to improve this model!

Volatility Regimes

There are multiple different ways in which we can define volatility regimes.
In this article I will be distinguishing between 3 different regimes: Low, Medium and High.

One very straightforward way to do this is to use quantiles.

low_vol_threshold = data['vol'].quantile(1/3)
high_vol_threshold = data['vol'].quantile(0.9)

This means that the 33.3% of volatility data lies below the low_vol_threshold and 90% of the vol data lies below the high_vol_threshold.

So we will define every datapoint below the low_vol_threshold as a low vol environment, everything between low_vol_threshold and high_vol_threshold as a medium vol environment and everything above high_vol_threshold as a high vol environment.

Discrete Model

One straightfoward way to change our quoting depending on the volatility regime is to simply use 3 different quoting widths.

low_vol_spread = 0.0004 # 4bps
medium_vol_spread = 0.0008 # 8bps
high_vol_spread = 0.002 # 20bps


data['MM_bid'] = data['mid'] * (1 - high_vol_spread/2)
data['MM_ask'] = data['mid'] * (1 + high_vol_spread/2)

# Adjust for medium volatility
data.loc[data['vol'] <= high_vol_threshold, 'MM_bid'] = (
    data['mid'] * (1 - medium_vol_spread/2)
)
data.loc[data['vol'] <= high_vol_threshold, 'MM_ask'] = (
    data['mid'] * (1 + medium_vol_spread/2)
)

# Adjust for low volatility
data.loc[data['vol'] <= low_vol_threshold, 'MM_bid'] = (
    data['mid'] * (1 - low_vol_spread/2)
)
data.loc[data['vol'] <= low_vol_threshold, 'MM_ask'] = (
    data['mid'] * (1 + low_vol_spread/2)
)

data['MM_bid'] = data.groupby([data.index.date, data.index.hour, data.index.minute, data.index.second])['MM_bid'].transform('first')
data['MM_ask'] = data.groupby([data.index.date, data.index.hour, data.index.minute, data.index.second])['MM_ask'].transform('first')


You can see how volatility starts to increase around 00:00:20 and our model then switches to a higher quoting width.

We can make this more sensitive by making the lookback of volatility smaller:

data['vol'] = data['mid'].rolling("30S").std()

Note: I picked those values simply for easy visualization. In real market making you would be using way different values and not just choosing them at random but through trial and error.
You run your algo live with some parameters => Gather data (How are you performing in each regime, are you reacting quickly enough or too slow etc.) => Adjust parameters => Repeat

Continuous Model

If you want to quote continuously, meaning that you don’t just have 3 different quoting levels but instead adjust your quote width in-between the thresholds then it’s all about interpolation.

Let’s start with the simplest case:

Linear Interpolation

For the third one we need to define another point where we reach our maximum quoting width.
Let’s go for 45 in this example:

In practice you’d also want to pick a non-zero lower bound with non-zero y_0 for the first interval since you don’t want to quote at 0bps (or close to 0bps numerically).
This could give you a function that looks something like this if we use x_0 = 2 and y_0 = 1:

Let’s plug it into python:

def quoting_width(x):
    if x <= 2:
        return 0.0001
    elif x <= 11:
        return (0.0001*(11-x)+0.0004*(x-2))/(11-2)
    elif x <= 32:
        return (0.0004*(32-x)+0.0008*(x-11))/(32-11)
    elif x <= 45:
        return (0.0008*(45-x)+0.002*(x-32))/(45-32)
    else:
        return 0.002
data['width'] = data['vol'].apply(quoting_width)
data['MM_bid'] = data['mid'] * (1 - data['width']/2)
data['MM_ask'] = data['mid'] * (1 + data['width']/2)

data['MM_bid'] = data.groupby([data.index.date, data.index.hour, data.index.minute, data.index.second])['MM_bid'].transform('first')
data['MM_ask'] = data.groupby([data.index.date, data.index.hour, data.index.minute, data.index.second])['MM_ask'].transform('first')

Some other interpolation methods are the following:

Polynomial interpolation

Instead of just fitting a linear function like we just did you could also fit higher order polynomials like second order (quadratic) or third order (cubic) etc.

Spline Interpolation

The family of splines is gigantic but the most commonly used one is a cubic spline.

Sigmoid Interpolation (Logistic Function)

Like the name implies you fit a sigmoid function.

Linear Interpolation with Smoothing

Instead of having those harsh changes in the slope (our slope is actually piecewise flat now) we smooth those edges out using techniques like applying smoothing functions close to the edge.
